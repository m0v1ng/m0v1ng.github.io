新玩法

pydbg的相较于OD等传统调试器的优点在于能和python交互，方便写调试器脚本，于是我打算用pydbg写一个能自动输入flag，然后下断点，输出关键的比较数据的脚本，通过输出的关键数据可以进行爆破。

我先写了个简单的题目

为了爆破这个程序，我写了这个脚本（有问题待解决）

我原本想利用subprocess实现自动输入的，但是发现pydbg attach到这个程序后，subprocess就不能利用管道自动输入了，因为父进程易主了，subprocess失去了对程序的控制，所以需要手动输入flag。这与我一开始的想法相违背，但是我尝试了很多方法均以失败告终

虽然不能自动输入，但是如果程序是以读文件的方式写入flag的话就没有这个问题了，这样我们就能下断点读取某个内存的值实现爆破了

2021.3.29更新

利用断点+修改EIP克服了自动输入的问题，可以进行爆破了，但是程序会输出很多字符，极大降低了爆破的效率，后面我连输出也绕过了，发现还是慢，调试器爆破就是慢吧

PythonGdb

PythonGdb使我们可以通过Python脚本来编写gdb调试脚本，详细文档 https://sourceware.org/gdb/wiki/PythonGdbTutorial

在gdb中用命令 source 即可导入 python 实现自动调试

这个重要性不必多说，官方文档 https://www.hex-rays.com/wp-content/static/products/ida/support/idapython_docs/ ，官方文档里的函数太多了，看起来很痛苦，国内似乎也没有很好的归纳文章，列一个勉强还可以的 https://wizardforcel.gitbooks.io/grey-hat-python/content/43.html
